    Team Project

    Query Engine for UBC Courses and Rooms
  • Team Member: Kohl Peterson

  • Scripts and Files: Html, JSON, TypeScript

  • Main Tasks: Query validation, Test construction, Query optimization (count, min, max, filter etc), Data filtration, Data parsing, Sending and receiving requests

  • Description: Upon receiving user input from the web server, the query engine is able to validate the query against the given EBNF specification, and produce a list of courses or rooms that satisfy the filtering condition. The schedule function within the query engine aims to produce a optimized pairing for course section, room and timeslot, while meeting additional constraints. The dataset for rooms are generated and parsed directly from the UBC website.
